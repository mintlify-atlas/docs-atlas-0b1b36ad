## Tool Capabilities

All tools route through a shared validation and capability-check layer that enforces:

- **Profile-based access control**: Configured via `[profiles]` in `settings.toml`
- **Runtime ceiling**: Global capability cap via `CURD_MODE` environment variable
- **Policy enforcement**: Filesystem boundaries, binary allowlists, and mutation gates

### Capability Atoms

| Tool | Canonical Operation | Capability Atom |
|------|---------------------|------------------|
| `search`, `find`, `contract` | Lookup | `lookup` |
| `graph`, `crawl`, `frontier` | Traverse | `traverse` |
| `read`, `doc`, `lsp` | Read | `read` |
| `edit`, `manage_file`, `mutate` | Change | `change.apply` |
| `workspace`, `session` | Session | `session.begin` / `session.verify` |
| `shell`, `build` | Exec | `exec.task` |

## Session Requirements

<Warning>
Mutating tools (`edit`, `manage_file`, `shell` with destructive commands) **require an active workspace session**. Call `workspace(action: 'begin')` before mutations.
</Warning>

### Workflow Example

```json
// 1. Open session
{"tool": "workspace", "action": "begin"}

// 2. Make changes
{"tool": "edit", "uri": "src/lib.rs::foo", "code": "...", "action": "upsert"}

// 3. Review changes
{"tool": "workspace", "action": "diff"}

// 4. Commit or rollback
{"tool": "workspace", "action": "commit"}
```

## Error Handling

All tools return structured JSON responses with an `error` field on failure:

```json
{
  "error": "Symbol 'foo' not found in workspace",
  "details": {}
}
```

Common error codes:
- `SESSION_REQUIRED`: Mutating operation without active session
- `STALE_STATE_CONFLICT`: Optimistic concurrency control failure
- `AST Churn Limit Exceeded`: Edit exceeds configured churn threshold
- `Validation failed`: Policy or capability check failure

## Index Management

Several tools depend on CURD's symbol index:

- **Automatic indexing**: `search` and `graph` auto-build indices on first use
- **Index modes**: `full`, `fast`, `lazy`, `scoped` (configured in `settings.toml`)
- **Cache invalidation**: Mutations automatically invalidate stale indices
- **Manual rebuild**: Use `curd index` CLI command

<Info>
Index build stats are returned in `search` responses under `index_stats`.
</Info>
